# experiments/ — доменные прогоны и проверки (часть требует закрытых данных/солвера)

Скрипты воспроизводят доменные результаты и проверки статьи. Самодостаточное ядро
(`../laziness.py`, `../baselines.py`, `../synthetic_demo.py`, `../sinkhorn_demo.py`)
работает на одном `numpy`; помеченные **[солвер]** скрипты зависят от пакетов
`python_forecast` / `python_solver` и данных Росстата, которые в публичный репозиторий
не входят. Здесь они — для провенанса (точный использованный код); численные CSV-сетки
карты применимости закоммичены. Ряд скриптов дополнительно требует `scipy`
(`pip install scipy`).

## Карта применимости (источник Рис. 4)

- `phase_map_struct.py` **[солвер]** — **каноническая структуросохраняющая карта
  Рис. 4**: `rho` = шум наблюдений уровней `N(t)`, из которого строится *валидная*
  балансовая `A(t)=build_A_phys(N_шум)`; min-norm восстановление, сетка `delta × rho`,
  60 начальных значений генератора. Пишет `phase_{null,ewma}_{ratio,win}.csv` (их
  читает `../figures/make_figures.py`). Выводы: null-сглаживание безопасно везде
  (ratio≤1, не зависит от `delta`), ewma вредит при большом `delta`/малом `rho`
  (до 1,8×). Прогон — `phase_map_struct.out`.
- `../phase_map.py` — самодостаточная (только `numpy`) абстрактная иллюстрация того
  же механизма на прямой матрице; пишет `*.abstract.csv` (в Рис. 4 не идут).

## Рынок труда (ОКВЭД-2, курированные данные 2017–2024)

- `rerun_curated_2024.py` **[солвер]** — **основной прогон**, дающий все числа
  Табл. 1/2/4 и текста §5–6 на курированной панели (8 снимков → 5 контрольных
  моментов 2020–2024): скользящий контроль + невязки баланса (Табл. 1); эталоны +
  масштабонезависимые метрики + батарея значимости с поправкой Холма (Табл. 2);
  инвариантность удержания, аблация «статический/динамический» (Табл. 4), корреляции
  энергия↔навыки, устойчивость по сетке солвера (§6). Вывод — `rerun_curated_2024.out`.
- `lambda_cv_real.py` **[солвер]** — причинный подбор `λ` на 8-снимковой панели
  (внутренний расширяющийся CV перестаёт быть вырожденным; оракул ≈ raw). Вывод —
  `lambda_cv_real.out`.
- `clustered_directional_M7.py` **[солвер]** — честная малосемпловая проверка
  единственного направленного результата (блуждание с дрейфом): кластеризация по
  годам вместо пула 41×5, год-кластерный бутстрап и перестановочный тест.
- `check_nonneg_M2.py` **[солвер]** — аудит: null-сглаживание не проецирует на
  неотрицательный конус; насколько компоненты уходят < 0 (≈1% на реальных панелях).
- `explore_strengthening.py` **[солвер]** — усиливающие расчёты: аблация
  статического/динамического члена, связь энергии траектории с надёжностью/MAE.
- `explore_laziness.py`, `explore_baselines.py`, `econ_significance.py` **[солвер]** —
  более ранние версии прогона лени / эталонов / значимости (провенанс; каноническими
  для чисел статьи являются `rerun_curated_2024.py` и `lambda_cv_real.py`).

## Физика (релятивистский электронный ансамбль)

- `physics_relaxation_rate.py` **[солвер]** — скорость релаксации *методом Улама*
  (парные микропереходы одних и тех же электронов cell→cell): SLEM `ρ*` даёт
  Λ_Улам≈0,68 против аналитической Λ=0,72 (отношение 0,95). Это **микротраекторные**
  данные, недоступные в постановке «два агрегированных снимка», — граница
  информативности.
- `recover_physics_gap.py` **[солвер]** — может ли *агрегатное* восстановление вернуть
  Λ количественно. Ответ: **нет** — щель одиночного восстановленного оператора
  неидентифицируема (при неизменной невязке ≈3,7·10⁻³ пробегает [0; 3,2Λ]; dim ker A=137).
  Физический экземпляр центральной теоремы о неидентифицируемости.

## Прочее

- `math_checks.py` **[солвер]** — числовая проверка утверждений о размерностях §3:
  полный строчный ранг `p`, dim ker A = 2n²+n, наименьшее ненулевое σ⁺_min
  (обусловленность на многообразии, §6).
- `probe_data_rate.py`, `probe_linear_response.py` — вспомогательные зонды.
- `timing.log` — журнал времени прогонов (всё CPU; GPU не требуется).

Канонические версии доменных скриптов живут в
`FindProbability/python_forecast/scripts/`. Запуск (из `FindProbability/`):

```bash
python -m python_forecast.scripts.explore_laziness
python -m python_forecast.scripts.explore_baselines
```

Для полностью публичного воспроизведения планируется заменить чтение закрытых данных
синтетическим генератором либо приложить обезличенные ряды `P̃` (CSV).
