# Восстановление оператора переноса мультикластерной незамкнутой системы по агрегированным данным: принцип наименьшего действия на его траектории

[![DOI](https://zenodo.org/badge/DOI/10.5281/zenodo.21140416.svg)](https://doi.org/10.5281/zenodo.21140416)

Код к одноимённой статье. Внутреннее рабочее название репозитория — «Behavioral
Laziness» / «Лень по поведению»; в тексте статьи этот же приём называется *инерцией*
правила эволюции (терминология сведена к «инерции», имя репозитория сохранено для
непрерывности DOI).

Рассматривается обратная задача восстановления оператора переноса мультикластерной
незамкнутой системы по двум последовательным агрегированным снимкам её состояний.
Задача существенно недоопределена: множество согласованных с наблюдениями операторов
образует аффинное многообразие, на котором тихоновская регуляризация с нулевой
априорной оценкой выделяет решение минимальной нормы — наиболее равномерное, близкое к
максимуму энтропии и потому систематически занижающее удержание, что неверно для
инерционной системы. В качестве не привязанного к предметной области априорного условия
предлагается **принцип наименьшего действия на траектории оператора**: правило эволюции
системы меняется во времени минимально. Реализованы два причинных способа его задания —
жёсткий (сохраняет наблюдаемый баланс `A·P` точно за счёт координации в нулевом
пространстве матрицы баланса) и мягкий (экспоненциальное сглаживание оператора), — а
также энтропийные baseline'ы (Синкхорн), карта применимости и причинный выбор параметра
регуляризации `λ`.

Репозиторий содержит самодостаточное (только `numpy`) ядро метода и воспроизведение
синтетического примера из статьи. Полные доменные прогоны (рынок труда РФ,
релятивистский электронный ансамбль) опираются на отдельные пакеты `python_forecast` /
`python_solver` и закрытые данные; в публичный репозиторий входят это ядро и синтетика,
а доменные результаты приводятся ссылкой или отдельным релизом.

## Состав

- `laziness.py` — причинная регуляризация траектории трансфер-оператора:
  `null_space_project`, `smooth_series_nullspace` (жёсткая, сохраняет `A·P`
  точно), `smooth_series_ewma` (мягкая), `trajectory_energy`.
- `baselines.py` — OT/энтропийные baseline'ы: `sinkhorn` (энтропийный
  оптимальный транспорт), `coupling_retention`, `transport_cost`.
- `synthetic_demo.py` — §5.3: контролируемая истина, вращающиеся `A(t)`; null
  помогает, ewma при реальном дрейфе вредит.
- `sinkhorn_demo.py` — §2.3/§3.4: энтропийный OT-коупл; churn = настраиваемый
  выбор (`reg`) → OT параметризует приор, а не устраняет.
- `phase_map.py` — §5.3: самодостаточная (только `numpy`) абстрактная иллюстрация
  карты применимости (delta × rho); null-space безопасна (ratio≤1, не зависит от
  тренда), ewma — зона вреда. Пишет `*.abstract.csv`. **Каноническая** карта Рис. 4
  строится структуросохраняющим прогоном `experiments/phase_map_struct.py`
  (`rho` — шум наблюдений уровней `N`, валидная балансовая `A`; нужен солвер); её CSV
  (`phase_*_ratio.csv`, `phase_*_win.csv`) закоммичены в `experiments/`.
- `lambda_cv.py` — §4.3: причинный выбор λ (inner-CV) бьёт raw и ~совпадает с
  оракулом для null-space.
- `experiments/` — доменные прогоны (нужны закрытые данные/солвер) + CSV-сетки
  карты + `timing.log`.
- `figures/make_figures.py` — генераторы рисунков 1–4 статьи (двуязычные, PNG+PDF;
  `numpy`+`matplotlib`); подробности — в `figures/README.md`.
- `tests/` — модульные тесты ядра (`test_laziness.py`, `test_baselines.py`),
  проверяющие инварианты из статьи (точное сохранение баланса, ортопроектор на
  ядро, причинность, монотонность удержания по ε у Синкхорна).
- `requirements.txt` — рантайм-зависимости (`numpy`);
  `requirements-dev.txt` — доп. инструменты (`pytest`, `matplotlib`).

## Запуск (самодостаточные, только numpy)

```bash
pip install -r requirements.txt
python synthetic_demo.py      # §5.3 механизм
python sinkhorn_demo.py       # §2.3 OT параметризует приор
python phase_map.py           # §5.3 карта применимости (CSV в experiments/)
python lambda_cv.py           # §4.3 причинный выбор lambda
```

Все детерминированы по сидам. Время прогонов — в `experiments/timing.log`
(всё CPU, < 10 с каждый; GPU не нужен).

## Тесты

```bash
pip install -r requirements-dev.txt
pytest -q                     # 37 тестов, ~0.2 с
```

Тесты кодируют математические гарантии из статьи: `null_space_project` —
ортогональный проектор (`A·Π(v)=0`, идемпотентность, симметрия), устойчивый и к
вырожденной `A`; `smooth_series_nullspace` сохраняет наблюдаемый баланс до
машинной точности при любом `λ`, причинна и не увеличивает энергию траектории;
`smooth_series_ewma` баланс не сохраняет (контраст с нуль-сглаживанием); Синкхорн
даёт корректные маргиналы, а удержание монотонно падает с ростом ε (механизм
Рис. 1). Данные для доменных таблиц не требуются — проверяется сам механизм.

## Рисунки

```bash
pip install -r requirements-dev.txt
python figures/make_figures.py ru    # или en
```

Генерирует `ris1`–`ris4` (Рис. 1–4 статьи). Если рядом есть дерево `article/`,
пишет в `article/figures[_en]`; в автономном клоне `code/` — в `figures/out/`.
`ris4` использует CSV из `experiments/` (при отсутствии — сначала
`python phase_map.py`). Подробности — в `figures/README.md`.

## Версии и цитирование

Актуальная версия — **1.2.0**. Постоянный (concept) DOI —
[10.5281/zenodo.21140416](https://doi.org/10.5281/zenodo.21140416) — всегда указывает
на последний релиз; у каждой версии есть собственный DOI. Как цитировать — см.
[`CITATION.cff`](CITATION.cff).

**Изменения в 1.2.0:**
- каноническая карта применимости (Рис. 4) стала *структуросохраняющей*
  (`experiments/phase_map_struct.py`: `rho` — шум уровней `N`, из него строится
  валидная балансовая `A`); прежняя абстрактная карта вынесена в `*.abstract.csv`;
- физика переатрибутирована: отношение 0,95 — оценка *методом Улама* по парным
  микропереходам (`physics_relaxation_rate.py`), а щель *агрегатного* восстановления
  неидентифицируема (при неизменной невязке пробегает [0; 3,2Λ],
  `recover_physics_gap.py`) — физический экземпляр теоремы о неидентифицируемости;
- курированная панель ОКВЭД-2 2017–2024, 5 контрольных моментов
  (`rerun_curated_2024.py`, `lambda_cv_real.py`): непревзойдённым остаётся наивный
  прогноз, сдвиг надёжности статистически незначим;
- добавлены проверки: `math_checks.py` (dim ker A = 2n²+n),
  `clustered_directional_M7.py`, `check_nonneg_M2.py`, `explore_strengthening.py`.
